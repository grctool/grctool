## Open Decisions / Flags (require maintainer follow-up)

- **F1 — Providers built but not wired at runtime.** AccountableHQ, GDrive, and
  the provider registry are implemented, but no code calls `RegisterFactory`, so
  `InitGlobalRegistry` registers zero providers and `grctool provider status`
  shows none. Decide whether to wire the factories (activate) or keep inert.
- **F2 — Product-boundary divergence (vs Vision/PRD).** Vision
  (`00-discover/product-vision.md`) and PRD (`01-frame/prd/requirements.md`)
  fence multi-provider / system-of-record as **roadmap, not current product**.
  The in-tree code (registry, AccountableHQ + GDrive SyncProviders, sync
  orchestrator, `index list`/`provider status`/`sync status` surfaces) has
  materially crossed that line — though it is not yet activated (F1). Decide
  whether to move the "Current Product Boundary" text or keep the providers
  gated as roadmap WIP. Per authority order this is a Vision/PRD decision, so it
  was **not** changed unilaterally.
- **F3 — Shipped commands without spec coverage.** `evidence download` (merged
  this cycle: on-demand Tugboat evidence retrieval, provenance, source
  collection) has no FEAT spec. Also `index`, `provider`, `sync status`,
  `terraform-index`, `validate-data`, `migrate`, `update`, `agent-context`.
  Decide: add specs (e.g., a FEAT-011 "Evidence Retrieval & Archive" / extend
  FEAT-010) or confirm intentionally unspecified.
- **F4 — HELIX process docs depend on removed `bd`/beads.** BEADS.md, EXECUTION.md,
  conventions.md, QUICKSTART.md, and the `actions/` (reconcile-alignment, check)
  prescribe a live `bd` tracker that no longer exists in this repo. Decide:
  reinstate beads, or rewrite the HELIX process docs to a tracker-less model.
